I Ran the Same AI Security Audit 25 Times. Here's What I Found.

March 3, 2026 · 5 min read · Shannon Agarwal
0xLLM
100% Detection Rate
25/25 Full-Sweep Runs
16.6% Token Variance
9 Vulnerabilities Tracked

Classic tools like Nessus or Nmap are deterministic — if a vulnerability exists, they report it every time. AI works differently. It generates responses probabilistically, meaning the same code can produce different findings on different runs.

I built a vulnerable Flask app with 9 documented security flaws — hardcoded credentials (CWE-798) and broken authentication (CWE-287, CWE-330, CWE-208, and others) — then sent it to the Claude API 25 times, byte-for-byte identical input each run.

The Result

100% detection across all 25 runs. Every vulnerability, every time.

But here's the more interesting finding: output token count varied from 1,167 to 1,376 tokens across identical runs. A 16.6% swing in response length on the exact same input. Non-determinism was real and measurable. It just showed up as verbosity variance, not detection failure.

Why the variance? My test bugs were syntactically obvious. A hardcoded string like sk_live_4eC39… doesn't require reasoning — it's pure pattern recognition. Non-determinism showed up as response length variance: the AI wrote 16.6% more tokens on its longest run than its shortest, covering the same bugs with different levels of detail.

Where This Gets Interesting

The real risk is subtler bugs: a timing attack buried in business logic, an IDOR that requires understanding your data model, an auth bypass that only triggers under specific conditions. Those require the model to reason, not just match patterns.

That's where token sampling variance becomes detection variance — where a critical vulnerability appears in 18 of 25 runs and you have no idea which run caught it.

The Takeaway for Security Teams

  • A single AI audit pass isn't enough for logic-level vulnerabilities
  • The right number of passes depends on how subtle your codebase's bugs are
  • A bug detected 80% of the time is not the same as one detected 100% of the time
  • You can't tell the difference with one run

This isn't an "AI is bad" take. It's a "know your tools" take.
