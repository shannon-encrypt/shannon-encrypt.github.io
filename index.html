LLM Audit Non-Determinism Study

Ran the same AI security audit 25 times on identical code to measure non-determinism in LLM-based vulnerability detection. Built a vulnerable Flask app with 9 documented flaws including hardcoded credentials (CWE-798) and broken authentication (CWE-287, CWE-330, CWE-208) then sent it to the Claude API 25 times with byte-for-byte identical input each run.

100% Detection rate across all 25 runs
16.6% Token count variance on identical input

Every vulnerability was caught every time, but non-determinism showed up as verbosity variance: 1,167 to 1,376 tokens per run on the same input. The test bugs were syntactically obvious, pure pattern recognition. The real risk is subtler bugs: timing attacks buried in business logic, IDORs that require understanding your data model, auth bypasses that only trigger under specific conditions. That's where token sampling variance becomes detection variance, where a critical vulnerability appears in 18 of 25 runs and you have no idea which run caught it. A single AI audit pass is not enough for logic-level vulnerabilities.
